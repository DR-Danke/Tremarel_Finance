Purpose & How to Use This Document

🌅 Morning (Before Service)

1. Receive WhatsApp daily summary

The system sends a morning message listing any pending or overdue tasks assigned to the owner. Format: personalized greeting with task count and descriptions.

2. Open Dashboard Dashboard

The dashboard greets the owner with a time-appropriate message ("Buenos dias", "Buenas tardes", or "Buenas noches") and a briefing summary. The owner scans:

  • Stat cards: Employee count, resource count (with low stock indicator), active documents, tasks completed today, overdue tasks (red highlight when > 0)
  • Alerts panel: Document expirations, low stock items, profitability alerts
  • Today's tasks: Tasks grouped by responsible person
  • Quick actions: "Nueva Tarea" and "Registrar Movimiento" buttons

3. Check profitability alerts Recetas

If any recipe's margin dropped below 60%, the dashboard shows a profitability alert. Navigate to Recetas to review cost vs. sale price and decide whether to adjust pricing or source cheaper ingredients.

4. Review document expirations Documentos

If the alerts panel shows expiring documents, delegate renewal to the compliance officer or navigate to Documentos to check details.

☀️ Midday (During Service)

5. Spot-check inventory Recursos

If a low stock alert appeared, check Recursos/Inventario for current stock levels and recent movements. Contact a supplier or instruct the manager to place an order.

6. Monitor task completion Dashboard

Return to the Dashboard to see how many tasks have been completed today vs. what remains pending.

🌙 End of Day

7. Review recipe margins Recetas

Scan the profitability column. Recipes with margin below 60% show a red "No Rentable" badge. Click any recipe to open the detail drawer showing an ingredient cost distribution pie chart. Adjust sale prices or flag recipes for review.

8. Check recent inventory movements Dashboard

The "Recent Inventory Movements" table shows the day's entries and exits for a quick audit trail.

📅 Weekly Activities

Review all document statuses

Filter Documentos by "Por Vencer" and "Vencido" to ensure nothing critical is missed.

Analyze resource costs

Review "Ultimo Costo" column in Recursos to identify cost trends across ingredients.

Staff review

Check Personas for completeness of employee records (WhatsApp numbers, roles) to ensure notifications reach everyone.

📋

Persona 2: Restaurant Manager

The operational backbone — coordinates daily tasks, manages inventory flows, ensures the team executes smoothly. On-site daily.

🌅 Morning (Before Service)

1. Receive WhatsApp daily summary

Morning message with the manager's own pending and overdue tasks.

2. Open Dashboard Dashboard

  • Today's tasks (left panel): All tasks for the day, grouped by responsible person. Overdue tasks highlighted in red.
  • Low stock items: Ingredients needing restocking before service.
  • Recent movements: Check if yesterday's expected deliveries were registered.

3. Assign and create tasks Eventos/Tareas

  • Create new tasks ("Tarea" type) for the day, assigning them to specific employees
  • Set notification channel to WhatsApp so employees receive their assignments
  • Review and reschedule any overdue tasks from previous days
  • Use the weekly calendar view to visualize the week's workload at a glance

4. Register morning deliveries Recursos

  • Click "Registrar Movimiento" → select resource, type "Entrada", reason "Compra"
  • Enter quantity and associate the supplier as the responsible person
  • Stock levels update automatically; low stock alerts resolve if stock rises above minimum
☀️ Midday (During Service)

5. Quick-complete tasks Eventos/Tareas

As employees finish tasks, use the checkmark icon to mark them completed. The Dashboard's "Tareas Completadas Hoy" counter updates in real time.

6. Handle stock issues Recursos

  • Check the resource to verify current stock
  • Register waste/spoilage as exit movement with reason "Merma"
  • If stock drops below minimum, a low stock alert is automatically generated

7. Register inventory exits Recursos

  • Type "Salida", reason "Uso" for general consumption
  • Type "Salida", reason "Merma" for waste or spoilage
🌙 End of Day

8. Task completion review Eventos/Tareas

Filter by today's date and status "Pendiente" to identify incomplete tasks. Follow up or reschedule.

9. Inventory reconciliation Recursos

Compare physical counts against system stock. Register adjustments with reason "Ajuste".

10. Plan tomorrow Eventos/Tareas

Create recurring tasks or one-time events for the next day.

👨‍🍳

Persona 3: Kitchen Staff / Chef

Interacts primarily through WhatsApp notifications and brief system check-ins. Focus: executing recipes, managing ingredient usage, completing tasks.

🌅 Start of Shift

1. Receive WhatsApp daily summary

Morning message listing today's tasks: prep work, production targets, cleaning checklists, etc. Overdue tasks from previous days are flagged.

2. Check Dashboard Dashboard

  • Today's tasks: Personal assignments with times and descriptions
  • Low stock alerts: Know which ingredients are running low before starting prep

3. Review recipe details Recetas

Click on today's recipes to see ingredient list with quantities per portion, current cost, and margin.

🔥 During Service

4. Produce recipe portions Recetas

  • Click the play icon on a recipe → enter number of portions
  • System automatically deducts ingredient quantities from inventory
  • If any ingredient drops below minimum stock, a low stock alert is generated

5. Report stock issues Recursos

Check current stock in Recursos/Inventario. Notify the manager to register the appropriate movement or place an order.

6. Complete tasks Eventos/Tareas

Use the quick-complete checkmark as prep tasks, cleaning tasks, or checklists are finished.

🌙 End of Shift

7. Final task check

Verify all assigned tasks for the shift are marked complete.

8. Report waste

Inform the manager of any spoilage or waste so they can register "Merma" exit movements.

Ensures the restaurant's legal documents, permits, and licenses are current. Works primarily with documents and expiration alerts.

🌅 Morning

1. Check expiration alerts Dashboard

  • Red alerts (URGENTE): Documents expiring within 7 days — require immediate action
  • Yellow alerts (Atencion): Documents expiring in more than 7 days — plan renewal
  • WhatsApp/email notifications also arrive with urgency formatting

2. Review Documentos page Documentos

  • Filter by status "Por Vencer" to see approaching expirations
  • Filter by status "Vencido" to see already-expired documents
💼 Core Work

3. Initiate renewals

Note document type, associated person, and expiration date. Contact the relevant authority or vendor. Track progress via Eventos/Tareas.

4. Register new documents Documentos

Edit existing document to update the expiration date, or add a new document with file upload. The system automatically creates new expiration alert events.

5. Manage permit presets Documentos

When adding permit-type documents, use pre-configured presets with automatic alert schedules:

6. Create compliance tasks Eventos/Tareas

Create tasks of type "Vencimiento" or "Tarea" to assign renewal responsibilities to specific people, with appropriate notification channels.

📅 Weekly / Monthly Activities

Full document audit

Review all documents across all types (Contrato, Permiso, Licencia, Certificado, etc.) to ensure completeness.

Verify person-document associations

Cross-reference Personas with Documentos to ensure employee-linked documents (e.g., food handling certificates) are up to date.

Report to owner

Summarize compliance status. The Dashboard's "Documentos Activos" stat card provides a quick count.

WhatsApp as an Operational Channel

8.1 Current Outbound Capabilities (Implemented)

RestaurantOS uses WhatsApp as a primary notification channel, reflecting the real-world communication habits of small restaurant teams in Latin America.

Architecture: NotificationService → WhatsAppAdapter (currently in stub mode, ready for Twilio or Meta Cloud API integration). Messages are capped at 4,096 characters. Phone numbers must include country code (e.g., +573001234567).

Message Type Trigger Format
Daily task summary POST /api/notifications/send-daily-summaries Personalized greeting, task count, numbered task list with times, overdue flags
Document expiration alert POST /api/notifications/process-expiration-alerts Urgency indicator (red ≤7 days, yellow >7 days), document name, days remaining
Low stock alert Automatic when current_stock < minimum_stock Resource name, current vs. minimum stock levels
Profitability alert Automatic when recipe margin drops below 60% Recipe name, cost, sale price, margin percentage
Event dispatch POST /api/notifications/dispatch or /dispatch-all Event-specific notification based on type and channel

8.3 Implementation Considerations

Webhook endpoint POST /api/notifications/whatsapp/webhook to receive inbound messages
Message parsing Simple keyword-based command router (no NLP or multi-turn conversations)
Authentication Match incoming WhatsApp number to a Person record's whatsapp field to resolve identity and restaurant context
Security Only registered persons with matching WhatsApp numbers can interact
Conversation model Stateless command-response pattern (no session tracking)
Provider support Both Twilio and Meta Cloud API support incoming webhook callbacks

System-Generated Daily Rhythm

RestaurantOS creates a natural operational cadence through automated notifications and alerts.

🌅 Morning Notifications

  • Daily task summaries to all employees with pending/overdue tasks (WhatsApp, Email)
  • Document expiration alerts to persons associated with expiring documents
  • Due event dispatch to all persons with events due today (per event's configured channel)

Triggered via: send-daily-summaries, process-expiration-alerts, dispatch-all

⚡ Event-Driven Alerts (Any Time)

  • Low stock: When current_stock < minimum_stock after any inventory movement
  • Profitability: When recipe margin drops below 60% after cost recalculation
  • Document expiration: When a document approaches its expiration date

Appear on Dashboard alerts panel + WhatsApp notification

🚨 Overdue Flagging

  • Events past scheduled date with status "Pendiente" are flagged as "Vencido"
  • Red background on Eventos/Tareas page
  • "VENCIDA" indicator in daily summaries
  • Dashboard groups overdue tasks for immediate attention
