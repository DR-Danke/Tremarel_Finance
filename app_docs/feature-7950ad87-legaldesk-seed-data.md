# Legal Desk Seed Data for Development

**ADW ID:** 7950ad87
**Date:** 2026-03-04
**Specification:** specs/issue-139-adw-7950ad87-sdlc_planner-legaldesk-seed-data.md

## Overview

Adds an idempotent SQL seed script for the Legal Desk module that populates all core `ld_` tables with realistic development data. The script enables developers to test repositories, services, and the specialist assignment engine's domain/jurisdiction matching logic without manual data entry.

## What Was Built

- Seed data SQL script covering 9 of 11 Legal Desk tables (documents and scores intentionally omitted)
- 3 clients (2 companies, 1 individual) across Colombia and Mexico
- 5 specialists with varied legal domains, experience levels, and hourly rates
- 12 specialist expertise entries spanning corporate, IP, labor, tax, and litigation
- 9 specialist jurisdiction entries across Colombia, Mexico, USA, and Spain
- 2 cases at different pipeline stages (new, in_progress) with different complexity levels
- 2 case-specialist assignments (proposed and active statuses)
- 6 case deliverables across pending, in_progress, and completed statuses
- 5 case messages (client, specialist, and internal types)
- 2 pricing history records (initial_quote and negotiation actions)

## Technical Implementation

### Files Modified

- `apps/Server/database/seed_legaldesk_data.sql`: New file — complete seed data script with 152 lines of SQL

### Key Changes

- All INSERTs wrapped in a `BEGIN; ... COMMIT;` transaction for atomicity
- Uses `ON CONFLICT DO NOTHING` on every INSERT for safe re-runnability
- Resets sequences via `setval()` after each table insert to prevent ID conflicts on future inserts
- Inserts follow strict foreign key dependency order: clients → specialists → expertise → jurisdictions → cases → case_specialists → deliverables → messages → pricing_history
- Data designed specifically for assignment engine testing: specialists have overlapping but distinct domain expertise and jurisdictions

## How to Use

1. Ensure the Legal Desk schema exists by running `create_legaldesk_tables.sql` first
2. Execute the seed script against your PostgreSQL database:
   ```bash
   psql $DATABASE_URL -f apps/Server/database/seed_legaldesk_data.sql
   ```
3. The script is idempotent — run it multiple times without errors
4. Query any `ld_` table to verify data was inserted

## Configuration

No additional configuration required. The script uses explicit IDs and EUR as the default currency per schema defaults.

## Testing

- Validate SQL syntax: `python -c "import sqlparse; stmts = sqlparse.parse(open('apps/Server/database/seed_legaldesk_data.sql').read()); print(f'Parsed {len(stmts)} statements')"`
- Run against a test database and verify record counts match: 3 clients, 5 specialists, 12 expertise, 9 jurisdictions, 2 cases, 2 assignments, 6 deliverables, 5 messages, 2 pricing records

## Notes

- `ld_case_documents` and `ld_specialist_scores` tables are intentionally not seeded — documents require actual file URLs, and scores should be generated by the scoring engine after case completion
- Case numbers use the `LD-YYYYMM-NNNN` format defined in the schema documentation
- All monetary values default to EUR currency
- The seed data supports assignment engine testing scenarios with overlapping specialist domains and jurisdictions for best-match scoring validation
