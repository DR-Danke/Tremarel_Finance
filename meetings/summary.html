Discussion Points

1. Pricing Engine & Margin Validation Logic

Extensive discussion on how profitability (%) and utility (absolute value) interact. Current system takes the nominally higher of the two, but both must be met or the quotation goes to approval. Key decision: Change UX so the user edits the final client price, and the system auto-calculates and validates both margin checks. If both pass, proceed; if either fails, require approval.

2. Vehicle Type Configuration

Vehicle types need corrections: remove weight labels to avoid confusion between sub-types (Minimula S2/S3, Tractomula variants). Missing vehicle type "Poderosa" (intermediate between Minimula and Tractomula). Weight capacities need correction (NHR: 2t, Turbo: 4.5t, Sencillo: 5–8.5t). Invalid "MULA" type in Excel template must be fixed.

3. Route Classification (Urban / National / Container Return)

System needs a route type field to distinguish between urban, national, and container return operations. Urban routes don't have CICETAC regulation. This classification affects which cost sources are available and which profitability policies apply per vehicle type.

4. Cost Data Sources & Update Frequency

Three cost sources: CICETAC (regulated floor), RNDC (market data via Gaussian statistical analysis), and Historical data. RNDC currently uses month-old data. Goal is daily updates. Buenaventura route (~80% of quotations) is most volatile and may need manual handling initially. Recommended: use the 85th percentile of dispatch volume for optimal pricing.

5. Quotation Amendment Functionality

Most structurally significant change requested. Need ability to modify accepted quotations when operational conditions change (e.g., container return location changes). Amendments should be permission-controlled. Especially important for spot quotations where client instructions change after approval.

6. Client Approval Workflow

Quotations sent via email with PDF attachment and QR code. Client can accept, reject, or request advisor. Need WhatsApp delivery and email-based one-click approval buttons since clients often approve informally. Internal approval queue managed by Carlos and Juan David.

7. Multi-Delivery & Container Return Scenarios

Complex scenarios (e.g., Buenaventura–Bogota with container return to Tocancipa) resolved via SAT (additional services). Container returns for different shipping lines (Evergreen, CMA) have different costs. ITR operations handled as SATs, not routes.

8. Price Rounding

Client-facing prices must be rounded up to the next multiple of 10,000 COP. Currently not implemented — system shows exact calculated values.

9. Postar TMS Integration (Phase 2)

Future phase: approved quotations and tariffs should sync to Postar TMS for service orders and manifests. Currently operates independently. Service order creation is the next development phase.

10. Quotation Cloning & Validity Periods

Clone feature works but needs to carry full configuration. Validity defaults to 15 days but should be configurable (weekly for volatile routes, annual for established clients).

11. Email Domain & Branding

Current emails from temporary "huevos" domain — unprofessional for client communications. Need proper company domain, letterhead on PDFs, and user-specific email signatures.
